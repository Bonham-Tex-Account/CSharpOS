Goal: Create a emulated Operating System in C# 

Current Architecture:

Operating System and Hardware are both mock structures that mimick real components:
-both will for starters be one class
-will communicate while running processes
-will be specified based on Operating system used (may make multiple OS classes later on)

Operating System: will mimick the operating system
-will handle swaping processes and communicate with the hardware on allocation and running
-Scheduler: will be part of the Operating System and live within it
--will context swap the processes in order to simulate a real computer
--will use a primitive method where each process is swapped after 5 instruction runs (if others exist)
--will keep track of the current process being used
-will recieve contact from Hardware about violations by processes or required actions and will respond accordingly(suspend process, crash, or request user to run restricted action)
-will crash and log processes that violate rules without permissions

Hardware: will mimick the hardware on a computer
-stores "memory" of the computer 
-will protect memory of the operating system and protected space of processes in an immutable section 
-will split this general memory for each process so they may think that they are using the same spcae but they are using their own sections of memory
-will contain many general registers used to simulate real registers used by the processes
-will contact the operating system when potential violations to memory rules (trying to read/write outside of process memory space), potential early suspended process I/O instructions, and invalid instruction attempts(not allowed instructions or nonexistent) 

Processes will mimick programs running on the computer:
-will store data required for operation two sections (protectected and process space)
-protectected space will include the instruction pointer, current register state when suspended
-process space will store the stack for that specific process
-contains data on how much memory will be allocated for this process and where it is allocated once it begins running

Running timeline:
-hardware will load the operating sytem being used 
-operating system will communicate protected sections of memory being used by itself and rules for instructions that the hardware should notify Operating System
-after setup is complete "processes" will be ran on our computer and will be loaded by our OS/hardware
-while running the processess , they will be context switched based on OS rules and ran until completion
-will output data about what computer is doing for visualization of what is happening and happened
-will finish when all processes are run and will "turn off"
